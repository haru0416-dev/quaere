## Scoring rubric

**Pre-scoring gate:** run each candidate through the Red Flags checklist (`metaphor.md` → Anti-patterns). **2+ "yes" answers = disqualified, do not score.** Only survivors of the availability gate proceed to numeric scoring.

**Scale:** rate each criterion 1–5 (5 = best-in-class, 3 = adequate, 1 = fails), then multiply by weight.

| Group | Criterion | What to evaluate | Weight |
|---|---|---|---|
| **Core** | Metaphor strength | Plants a concrete image? Connects to function? | 3× |
| **Core** | Memorability | Remembered tomorrow after hearing once? | 3× |
| **Core** | Story | 15-sec origin story? "That's clever" reaction? | 2× |
| **Core** | Distinctiveness | Stands out from competitors? Confusable? | 2× |
| **Practical** | Phone test | Spell it after hearing / say it after reading? | 2× |
| **Practical** | Length | 1–2 syllables ideal, 3 ok, 4+ problematic; <7 chars ideal | 1× |
| **Practical** | Sound alignment | Sounds match product character (phonosemantics)? | 1× |
| **Practical** | Availability | Available on critical platforms? | 1× |
| **Context** | Brand fit | If part of a family, does it belong? | 1× |
| **Context** | Global friendliness | Pronounceable abroad? Negative meanings? | 1× |
| **Context** | Longevity | Still works in 5–10 yrs? Dates itself? | 1× |
| **Context** | Searchability | Googleable? Wikipedia/major-brand competition? | 2× |
| **Context** | Competitive density | Adjacent-space usage (5=none … 1=crowded) | 1× |

**Total out of 105** (weights sum to 21 × max score 5). Reading: **>75 strong, >85 exceptional**; the final checklist requires **scored above 70** to commit. Tiebreakers within 5 pts: stronger metaphor → cleaner phone test → better origin story → better availability → instinct.

## Contextual sentence test

Put each finalist into real sentences and read aloud:

- Introduction: "Have you tried `___`?" / "We just launched `___`."
- Explanation: "We built `___` to solve [problem]." / "`___` is a [category] that [does what]."
- Daily use: "Check the `___` dashboard." / "The `___` API is down."
- **Word of mouth (most important):** "You should try this thing called `___`." If it feels awkward spoken aloud, word-of-mouth growth is crippled.

## Worked example

The upstream case file documents etymologies, not full per-product pipelines. The two cases below are the only ones recording rejected candidates and/or availability work, so they best support a "function → territory → candidates → reject → availability → origin story" narrative. All facts are as documented upstream.

**Example 1 — Vercel (frontend deployment platform)**

- **Core function:** frontend hosting / deployment.
- **Territory chosen:** invented name grounded in morphemes (not metaphor or dictionary word).
- **Candidate rejected:** the company's original name **"ZEIT"** — hard to spell and pronounce across languages (fails phone test / global-friendliness).
- **Availability validation:** the replacement was **tested by linguists across 5 languages** for pronounceability before adoption.
- **Final name + origin story:** **Vercel** — invented, connecting to *versatile*, *accelerate*, *excel*; 2 syllables, globally pronounceable. A deliberate professional rename away from an unpronounceable original.

**Example 2 — Datadog (infrastructure monitoring)**

- **Core function:** monitoring.
- **Territory chosen:** compound name (two halves that each carry meaning).
- **Internal origin:** named after **"Datadog17,"** a feared Oracle database instance at the founders' previous company.
- **Why it survives anti-patterns:** both halves earn their place — "data" is literal, **"dog" adds watchdog personality** to a dry category; the internal origin gives authenticity. Contrast the failed *CloudWatch*: "Cloud" filler + "Watch" generic.
- **Final presentation:** **Datadog** — "the database instance everyone feared, turned into a watchdog for your whole stack." Two components, concrete, personable.

**Anti-pattern rejects to cite as the "fell out" column** (documented failures): *Uptime Robot* (descriptive + filler), *CloudWatch* (filler + generic), *StatusCake* (quirky, no metaphor grounding — "why cake?"), *Site24x7* (numbers date the name), *Basecamp* (metaphor too broad, went generic).
